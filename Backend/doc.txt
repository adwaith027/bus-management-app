What Django does at startup

You run:
python manage.py runserver

Django:
Loads settings.py
Reads INSTALLED_APPS

For each app in INSTALLED_APPS:
Django loads its AppConfig (from apps.py)
Creates an instance of that AppConfig

After all apps are loaded:
Django calls app_config.ready() once per app

You should not put irreversible logic there !!!!

apps.py exists to run app-level startup code when Django loads an app.
